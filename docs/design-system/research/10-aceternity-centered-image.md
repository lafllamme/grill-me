# 10: Aceternity centered-image hero

Source: https://ui.aceternity.com/live-preview/hero-with-centered-image

![Centered product hero](./images/10-aceternity-centered.png)

## Observed system

- A centered editorial headline leads directly into one oversized product image.
- The product frame uses nested radii around `20-32px`.
- The surrounding page is intentionally quiet; the framed product provides most visual detail.
- A soft shadow separates the frame without turning it into a floating card.

## Grillme translation

- Use the same direct sequence: promise, GitHub target control, roast stage.
- Make the roast interface wide enough to read as the product, not as a terminal widget.
- Use nested radius tiers for stage, output viewport, and controls.

## Avoid

The strictly centered SaaS hero is too generic by itself. Grillme needs asymmetric type and background tension to make the pattern ownable.

## Behavior and extractable components

- The product frame follows the promise immediately, minimizing the gap between explanation and evidence.
- Extract the nested stage-frame hierarchy for the result viewport.
- Keep Grillme's headline and support copy asymmetric instead of copying the centered composition.
